# 업데이트 가이드

배포 후 코드를 수정하고 업데이트하는 방법을 안내합니다.

## 📝 업데이트 프로세스

### 1단계: 로컬에서 수정 및 테스트

```bash
# 1. 개발 서버 실행하여 변경사항 확인
npm run dev

# 2. 브라우저에서 http://localhost:3000 접속하여 테스트
# 3. 모든 기능이 정상 작동하는지 확인
```

### 2단계: 빌드 테스트 (중요!)

```bash
# 빌드가 성공하는지 확인
npm run build

# 빌드된 결과 미리보기
npm run preview
```

빌드 오류가 없으면 다음 단계로 진행합니다.

### 3단계: GitHub에 업로드

```bash
# 변경사항 확인
git status

# 변경된 파일 추가
git add .

# 커밋 (변경 내용을 설명하는 메시지 작성)
git commit -m "문제 내용 수정" 
# 또는
git commit -m "UI 개선: 사이드바 폰트 크기 조정"
# 또는
git commit -m "새로운 퀴즈 추가"

# GitHub에 업로드
git push origin main
```

### 4단계: 자동 배포 확인

#### Vercel 사용 시
1. **자동 배포**: GitHub에 push하면 자동으로 배포가 시작됩니다
2. **배포 상태 확인**:
   - [vercel.com](https://vercel.com) 접속
   - 대시보드에서 프로젝트 선택
   - "Deployments" 탭에서 배포 상태 확인
   - ✅ "Ready" 상태가 되면 배포 완료
3. **배포 로그 확인**: 각 배포를 클릭하면 빌드 로그를 볼 수 있습니다

#### Netlify 사용 시
1. **자동 배포**: GitHub에 push하면 자동으로 배포가 시작됩니다
2. **배포 상태 확인**:
   - [app.netlify.com](https://app.netlify.com) 접속
   - 사이트 선택
   - "Deploys" 탭에서 배포 상태 확인
   - ✅ "Published" 상태가 되면 배포 완료

### 5단계: 배포된 사이트 확인

1. **배포 완료 후 1-2분 대기** (빌드 시간)
2. **배포된 URL 접속**하여 변경사항 확인
3. **브라우저 캐시 문제**가 있을 수 있으므로:
   - `Ctrl + Shift + R` (Windows) 또는 `Cmd + Shift + R` (Mac)으로 강력 새로고침
   - 또는 시크릿 모드로 접속

## 🔍 배포 상태 확인 방법

### Vercel 대시보드
```
1. vercel.com 접속 → 로그인
2. 프로젝트 선택
3. "Deployments" 탭 클릭
4. 최신 배포의 상태 확인:
   - 🟡 Building: 빌드 중
   - 🟢 Ready: 배포 완료
   - 🔴 Error: 오류 발생
```

### Netlify 대시보드
```
1. app.netlify.com 접속 → 로그인
2. 사이트 선택
3. "Deploys" 탭 클릭
4. 최신 배포의 상태 확인:
   - 🟡 Building: 빌드 중
   - 🟢 Published: 배포 완료
   - 🔴 Failed: 오류 발생
```

## 🚨 문제 해결

### 배포가 실패하는 경우

1. **빌드 로그 확인**
   - Vercel/Netlify 대시보드에서 빌드 로그 확인
   - 오류 메시지 확인

2. **로컬에서 빌드 테스트**
   ```bash
   npm run build
   ```
   - 로컬에서 빌드가 실패하면 배포도 실패합니다
   - 오류를 먼저 수정해야 합니다

3. **환경 변수 확인**
   - `GEMINI_API_KEY`가 올바르게 설정되어 있는지 확인

### 변경사항이 반영되지 않는 경우

1. **배포가 완료되었는지 확인**
   - 대시보드에서 "Ready" 또는 "Published" 상태인지 확인

2. **브라우저 캐시 삭제**
   - `Ctrl + Shift + Delete` (Windows) 또는 `Cmd + Shift + Delete` (Mac)
   - 캐시된 이미지 및 파일 삭제
   - 또는 시크릿 모드로 접속

3. **강력 새로고침**
   - `Ctrl + Shift + R` (Windows) 또는 `Cmd + Shift + R` (Mac)

## 💡 업데이트 팁

### 커밋 메시지 작성 가이드
명확한 커밋 메시지를 작성하면 나중에 변경 이력을 추적하기 쉽습니다:

```
✅ 좋은 예:
- "퀴즈 문제 내용 수정"
- "Pay-Off Matrix 문제 교체"
- "UI 개선: 사이드바 폰트 크기 증가"
- "새로운 학습 모드 추가"

❌ 나쁜 예:
- "수정"
- "업데이트"
- "asdf"
```

### 변경 전 체크리스트

- [ ] 로컬에서 `npm run dev`로 정상 작동 확인
- [ ] `npm run build`로 빌드 성공 확인
- [ ] 주요 기능 테스트 (퀴즈 풀기, 정답 확인 등)
- [ ] 오타나 문법 오류 확인
- [ ] 커밋 메시지 작성
